## Build-app course mode

Review the script as a **build-along** episode about a **real application**.

### Balance check (mandatory)

Respect active Narrative balance:

| Mode | Practice | Theory |
|------|----------|--------|
| practice-first | ~65–75% | ~25–35% |
| balanced | ~50% | ~50% |
| theory-first | ~35% | ~65% |

Flag when timed `Theory` blocks dominate a practice-first episode, or when theory-first still lacks a real file walkthrough + hero demo.

### Educational priorities for build-app

1. Hook states what will be built and what success looks like on camera.
2. Theory **refreshes** context for the code — it must not replace the walkthrough.
3. File walkthrough teaches role → outcome → key imports/types/methods.
4. Cognitive load: one idea at a time; prefer showing the repo over abstract diagrams when both exist.
5. Dedup against Prior coverage / Concepts introduced — suggest callbacks, not re-lectures.
6. Demo must match the promised result (before/after chunking, API JSON, etc.).

Keep edits concrete (cut/merge which paragraphs, which file to expand).
